# 패킷트레이서 실습


시스코 장비의 CLI(Command Line Interface) 접속 단계

## 장비 접속 및 모드 전환 단계 요약

장비 부팅 후 설정 완료까지의 핵심 흐름입니다.

| 단계 | 모드 이름 | 프롬프트 상태 | 입력 명령어 | 비고 |
| --- | --- | --- | --- | --- |
| **Step 1** | **사용자 실행 모드** (User EXEC) | `Router>` | `enable` (또는 `en`) | 기본 조회만 가능 |
| **Step 2** | **관리자 실행 모드** (Privileged EXEC) | `Router#` | `configure terminal` (`conf t`) | 상세 확인 및 설정 진입 |
| **Step 3** | **전역 설정 모드** (Global Config) | `Router(config)#` | (다양한 설정 명령어) | 실제 환경설정 수행 |

---

## 1. 초기 부팅과 인터페이스 진입

장비에 전원을 공급하면 내부적으로 하드웨어 점검(POST)을 마치고 운영체제(IOS)를 로드합니다. 모든 과정이 끝나면 화면에 엔터를 누르라는 메시지가 뜹니다.

* **최초 질문:** 만약 `Would you like to enter the initial configuration dialog? [yes/no]` 메시지가 나온다면, 직접 설정을 위해 반드시 **`no`**를 입력합니다.

## 2. 사용자 실행 모드 (User EXEC Mode)

장비에 접속하자마자 만나는 첫 번째 관문입니다.

* **프롬프트:** `Router>` (꺽쇠 기호)
* **특징:** 보안을 위해 매우 제한적인 명령어만 사용 가능합니다. 장비의 설정을 바꿀 수 없는 '읽기 전용' 상태입니다.

## 3. 관리자 실행 모드 (Privileged EXEC Mode)

장비의 운영 상태를 자세히 들여다보고 설정을 저장하거나 삭제할 수 있는 권한을 가집니다.

* **프롬프트:** `Router#` (해시 기호)
* **진입:** 사용자 모드에서 `enable` 명령어를 입력하여 들어옵니다.
* **주요 작업:** `show` 명령어를 통해 현재 설정(`running-config`)이나 인터페이스 상태를 확인합니다.

## 4. 전역 설정 모드 (Global Configuration Mode)

장비의 이름을 바꾸거나 인터페이스에 IP를 넣는 등 실제 데이터가 흐르게 만드는 '환경설정'의 본론입니다.

* **프롬프트:** `Router(config)#`
* **진입:** 관리자 모드에서 `configure terminal` 명령어를 입력합니다.
* **특징:** 여기서 입력하는 명령어는 장비 전체의 동작에 즉시 영향을 미칩니다.
* **빠져나가기:** `exit`를 치면 한 단계 뒤로, `end` 혹은 `Ctrl+Z`를 누르면 즉시 관리자 모드(`#`)로 돌아갑니다.
